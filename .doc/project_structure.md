# Project Structure Guide

Welcome to the Digital Garden repository! This document outlines the physical architecture of the codebase, explaining what each root folder and system file does. Use this as a map to navigate the repository for future development.

> [!NOTE]
> This guide does not document the interior hierarchy of `src/site/notes/` as that serves as your living Obsidian vault.

## Root Directory

### System & Configuration

- **`.eleventy.js`** — The heart of the build engine. This configures Eleventy (11ty) and houses the custom `markdown-it` plugins responsible for successfully parsing Obsidian-specific syntax (callouts, links, Mermaid diagrams) into standard HTML.
- **`.eleventyignore`** — Lists directories that the Eleventy build engine should skip entirely (e.g. ignoring `src/site/notes/_drafts` so they don't appear on the live site).
- **`.env`** — The environment variable file controlling global site toggles (e.g. `dgShowFileTree`, `dgEnableSearch`) and feature flags initially established by the digital garden system.
- **`.gitignore`** — Critical security definitions. It prevents your massive local compiled plugin binaries and sensitive tokens (like GitHub keys) from being accidentally uploaded to your public repository.
- **`package.json` / `package-lock.json`** — The Node.js project manifests. They manage all the dependencies required to build the site and define your build scripts (like `npm run dev` and `npm run build`).

### Workspace & IDE

- **`.obsidian/`** — Your local Obsidian vault's workspace configuration. This folder houses your Obsidian theme files, hotkey mappings, layout workspace states, and community plugin code.
- **`.agent/`** — Contains automated AI and development workflows (e.g., `/commit`, `/dev`) to orchestrate routine GitHub or build tasks.
- **`.github/`** — Holds GitHub Actions workflows (e.g., `build.yml`) that automatically verify the health of the repository upon each push.
- **`.doc/`** — The "Documentation" directory containing administrative metadata about the project itself, such as the `refinement_log.md` and this structure guide.

### Auto-Generated Content (Do Not Edit)

- **`dist/`** — The final, minified HTML, CSS, and JS output generated by Eleventy. This is what Cloudflare Pages reads and serves to the world. Any manual changes here are wiped out and overwritten on the next build.

## The Source Code (`src/`)

The `src/` folder contains exactly how the website is generated, linking the raw text of your notes to the sophisticated frontend user interface.

### `src/helpers/`

Contains modular JavaScript utility scripts that do heavy data processing before it touches the frontend templates:

- **`bases-engine/`** — Resolves code blocks written for the custom "Bases" card rendering system.
- **`filetreeUtils.js`** — Builds the hierarchical JSON structure for the left sidebar file explorer.
- **`linkUtils.js`** — Parses raw `[[Obsidian Links]]` to valid web URLs.

### `src/site/`

The core frontend environment for the site.

- **`_data/`** — Eleventy global data files. These inject dynamic variables across every page.
- **`_includes/`** — The Nunjucks (`.njk`) template logic.
  - **`layouts/`** — High-level page wrappers. For example, `note.njk` determines the overarching structure of an article, while `index.njk` dictates the homepage structure.
  - **`components/`** — Granular UI chunks that inject into layouts. This includes `navbar.njk`, `sidebar.njk`, `graphScript.njk`, and the critical `lucideIcons.njk` system.
- **`styles/`** — The SCSS typography and aesthetic architecture.
  - **`_theme-control.scss`** — Your central design token command post for defining Tokyo Night colors, Atkinson fonts, and spacing variables.
  - **`custom-style.scss`** — The master sheet where local component styling overrides are written.
- **`img/`** — Static imagery or structural vector assets used by the core UI (not your actual note attachments).
- **`notes/`** — *The Vault itself.* This is where all of your living content and attachments reside.
